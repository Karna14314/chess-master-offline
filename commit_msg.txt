⚡ Optimize evaluation loops in SimpleBotService

💡 **What:**
- Replaced the string-based square lookup `board.get(_indexToSquare(rank * 8 + file))` with direct 0x88 array index access `board.board[rank * 16 + file]`.
- Replaced similar logic when checking king safety for pawn shields.

🎯 **Why:**
- The `_evaluatePosition` method loops 64 times inside a very tight loop (minimax alpha-beta tree search). Computing square string names (e.g. `'e4'`) and performing map lookups creates unnecessary GC pressure and computation overhead. Bypassing string generation and accessing the board array by index directly speeds up evaluation.

📊 **Measured Improvement:**
- Benchmark: `test/benchmarks/engine_evaluation_benchmark.dart`
- Baseline: 1657ms average execution time.
- Optimized: 1592ms average execution time.
- Improvement: ~65ms (~4% speedup in tree search performance)
